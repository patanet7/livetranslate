INFERENCE FLOW COMPARISON

Reference Greedy Decoding (beam_size=1):
  1. Encode audio once
  2. Decode tokens one at a time
  3. Select argmax token at each step (no beam ranking)
  4. Single KV-cache
  Total: Minimal overhead

Our Beam Search (beam_size=2):
  1. Encode audio once
  2. Decode 2 hypotheses in parallel
  3. Rank beams at each step (compute logprobs, sort)
  4. Double KV-cache memory
  Total: ~2x computational overhead
